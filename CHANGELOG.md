# Changelog

Todas as alterações relevantes deste projeto serão documentadas neste arquivo.

O formato deste documento é baseado em **Keep a Changelog** e o projeto segue o padrão de versionamento **Semantic Versioning (SemVer)**.

* **MAJOR**: mudanças incompatíveis com versões anteriores.
* **MINOR**: novas funcionalidades compatíveis.
* **PATCH**: correções de bugs e melhorias sem alteração de compatibilidade.
