1 – Introdução
1.1 Objetivo
Este documento apresenta o Manual do Usuário do Gestor de Territórios e Publicações (GTP).
Seu objetivo é orientar os usuários na utilização do sistema, explicando suas funcionalidades, fluxos operacionais e boas práticas de uso. O manual foi elaborado para servir como referência durante o uso diário da aplicação, reduzindo dúvidas e facilitando a adoção do sistema.

1.2 Finalidade
O Manual do Usuário tem como finalidades:
•	orientar o uso das funcionalidades do GTP;

•	padronizar os procedimentos operacionais;

1.4 Visão Geral do Sistema
O Gestor de Territórios e Publicações (GTP) é uma plataforma web desenvolvida para apoiar o gerenciamento das atividades relacionadas aos territórios, publicadores, designações, reuniões e publicações da congregação.
O sistema centraliza informações em um único ambiente, permitindo maior organização, controle e rastreabilidade das operações.
Entre as principais funcionalidades estão:
•	gerenciamento de usuários;
•	cadastro de publicadores;
•	controle de territórios;
•	controle de publicações;
•	designações e responsabilidades;
•	emissão de relatórios;
•	notificações;
•	indicadores gerenciais.

2 – Primeiro Acesso e Login
2.1 Objetivo
Este capítulo descreve o processo de acesso ao Gestor de Territórios e Publicações (GTP), desde o primeiro login até o encerramento seguro da sessão.
São apresentados os procedimentos para autenticação, recuperação de senha, alteração da senha inicial e as recomendações de segurança relacionadas ao acesso ao sistema.

2.2 Primeiro Acesso
O primeiro acesso ao GTP é realizado utilizando as credenciais fornecidas pelo administrador do sistema.
Normalmente, o usuário receberá:
•	endereço de acesso ao sistema;
•	nome de usuário ou e-mail;
•	senha temporária (quando aplicável).
Após o primeiro login, o sistema poderá solicitar a alteração da senha para garantir maior segurança.

2.3 Acessando o Sistema
Para acessar o GTP:
1.	Abra um navegador compatível.
2.	Informe o endereço (URL) do sistema.
3.	Aguarde o carregamento da tela de login.
Fluxo de acesso:
Navegador
      │
      ▼
Página Inicial
      │
      ▼
Tela de Login
A página de login será o ponto de entrada para todos os perfis de usuário.

2.4 Tela de Login
A tela de autenticação conterá os seguintes elementos:
•	campo Usuário ou E-mail;
•	campo Senha;
•	botão Entrar;
•	opção Esqueci minha senha;
•	mensagem de erro em caso de autenticação inválida.
Após preencher as credenciais corretamente, o usuário deverá selecionar Entrar.

2.5 Processo de Autenticação
O fluxo de autenticação ocorre da seguinte forma:
Usuário
    │
    ▼
Informar usuário e senha
    │
    ▼
Validação das credenciais
    │
    ▼
Autenticação realizada
    │
    ▼
Dashboard
Caso as credenciais estejam incorretas, o sistema exibirá uma mensagem informando que o acesso não foi autorizado.

2.6 Recuperação de Senha
Caso o usuário não se lembre da senha cadastrada, deverá utilizar a opção Esqueci minha senha.
Fluxo de recuperação:
1.	Informar o e-mail cadastrado.
2.	Confirmar a solicitação.
3.	Receber um link de redefinição.
4.	Criar uma nova senha.
5.	Efetuar novo login.
O link de redefinição terá validade limitada por questões de segurança.


2.7 Alteração da Senha Inicial
Quando o usuário acessar o sistema utilizando uma senha temporária, será solicitado que defina uma nova senha.
A nova senha deverá atender aos critérios estabelecidos pela política de segurança do sistema, como:
•	comprimento mínimo;
•	combinação de letras, números e caracteres especiais;
•	não reutilização de senhas recentes (quando configurado).
Após a alteração, a senha anterior deixará de ser válida.
2.8 Sessão do Usuário
Após a autenticação, será iniciada uma sessão vinculada ao usuário.
Durante a sessão, o sistema poderá:
•	controlar permissões de acesso;
•	registrar ações realizadas;
•	manter o usuário autenticado enquanto houver atividade.
Em caso de inatividade prolongada, a sessão poderá ser encerrada automaticamente por motivos de segurança.
2.9 Encerramento da Sessão (Logout)
Ao finalizar o uso do sistema, recomenda-se sempre realizar o logout.
Procedimento:
1.	Selecionar o menu do usuário.
2.	Clicar na opção Sair.
3.	Confirmar o encerramento da sessão, quando solicitado.
Fluxo:
Menu do Usuário
        │
        ▼
Botão Sair
        │
        ▼
Sessão Encerrada
        │
        ▼
Tela de Login
Essa prática evita acessos indevidos em dispositivos compartilhados.

2.10 Mensagens de Erro Mais Comuns
Durante o processo de login, algumas mensagens poderão ser exibidas.
Mensagem	Possível Causa
Usuário ou senha inválidos	Credenciais incorretas
Usuário inativo	Conta desabilitada pelo administrador
Sessão expirada	Tempo de inatividade excedido
Acesso não autorizado	Permissões insuficientes
Erro de conexão	Problema de comunicação com o servidor
Caso o problema persista, recomenda-se entrar em contato com o administrador do sistema.

2.11 Recomendações de Segurança
Para proteger a conta de acesso, recomenda-se:
•	não compartilhar usuário e senha;
•	utilizar senhas fortes;
•	alterar a senha periodicamente;
•	evitar utilizar a mesma senha em outros sistemas;
•	não salvar credenciais em computadores públicos;
•	realizar logout ao finalizar o uso.
Essas medidas reduzem significativamente o risco de acesso não autorizado.

2.12 Boas Práticas
Durante o acesso ao sistema, recomenda-se:
•	verificar se está utilizando o endereço oficial do GTP;
•	manter o navegador atualizado;
•	utilizar conexões seguras;
•	revisar as informações digitadas antes de confirmar o login;
•	comunicar imediatamente qualquer atividade suspeita ao administrador.

2.13 Perguntas Frequentes
Esqueci minha senha. O que devo fazer?
Utilize a opção Esqueci minha senha na tela de login e siga as instruções enviadas para o e-mail cadastrado.
Posso utilizar o mesmo usuário em mais de um computador?
Sim, desde que respeitadas as políticas de segurança definidas pelo administrador do sistema.
O sistema encerrou minha sessão automaticamente.
Isso pode ocorrer por inatividade ou por políticas de segurança. Basta realizar um novo login para continuar utilizando o sistema.

4 – Cadastro e Gerenciamento de Usuários
4.1 Objetivo
Este capítulo apresenta as funcionalidades relacionadas ao cadastro e ao gerenciamento de usuários do Gestor de Territórios e Publicações (GTP).
São descritos os procedimentos para criação, consulta, edição, ativação, desativação e administração das contas de acesso, bem como as boas práticas para manter a segurança e a organização do sistema.
Observação: As operações descritas neste capítulo estarão disponíveis apenas para usuários que possuam as permissões necessárias, como administradores ou responsáveis pela gestão do sistema.

4.2 O que é um Usuário?
Um usuário representa uma pessoa autorizada a acessar o GTP.
Cada usuário possui uma conta individual composta por:
•	nome completo;
•	usuário ou e-mail para acesso;
•	senha;
•	perfil de acesso;
•	status da conta;
•	informações de auditoria (data de criação, último acesso etc.).
Cada conta é pessoal e intransferível.

4.3 Acessando o Módulo de Usuários
Para acessar o gerenciamento de usuários:
1.	Efetue o login no sistema.
2.	No menu lateral, selecione Administração.
3.	Clique em Usuários.
Fluxo:
Dashboard
      │
      ▼
Administração
      │
      ▼
Usuários
Será exibida a lista de usuários cadastrados.

4.4 Tela de Gerenciamento de Usuários
A tela principal apresenta uma relação dos usuários cadastrados.
As informações normalmente exibidas incluem:
Campo	Descrição
Nome	Nome completo do usuário
Usuário/E-mail	Identificador utilizado no login
Perfil	Perfil de acesso atribuído
Status	Ativo ou Inativo
Último Acesso	Data e hora do último login
Ações	Visualizar, Editar, Ativar/Desativar
Dependendo do perfil do usuário autenticado, algumas ações poderão não estar disponíveis.

4.5 Cadastro de Novo Usuário
Para cadastrar um novo usuário:
1.	Clique em Novo Usuário.
2.	Preencha os dados obrigatórios.
3.	Defina o perfil de acesso.
4.	Informe os dados de autenticação.
5.	Clique em Salvar.
Os principais dados solicitados poderão incluir:
•	nome completo;
•	e-mail;
•	nome de usuário (quando aplicável);
•	telefone (opcional);
•	perfil;
•	congregação (quando aplicável);
•	status inicial.
Após a confirmação, o novo usuário será registrado no sistema.

4.6 Perfis de Acesso
Cada usuário possui um perfil que determina as funcionalidades disponíveis.
Exemplos de perfis:
Perfil	Permissões Gerais
Administrador	Acesso completo ao sistema
Superintendente de Serviço	Gerenciamento operacional da congregação
Ancião	Acesso às funções administrativas autorizadas
Servo Ministerial	Acesso conforme designações
Publicador	Acesso às funcionalidades permitidas para consulta e acompanhamento
As permissões são definidas pelo administrador e podem variar conforme as necessidades da congregação.

4.7 Consulta de Usuários
A lista de usuários poderá ser consultada utilizando:
•	pesquisa por nome;
•	pesquisa por e-mail;
•	pesquisa por usuário;
•	filtros por perfil;
•	filtros por status;
•	ordenação das colunas.
Esses recursos facilitam a localização rápida de qualquer usuário cadastrado.

4.8 Edição de Usuários
Para alterar os dados de um usuário:
1.	Localize o registro desejado.
2.	Clique em Editar.
3.	Atualize as informações necessárias.
4.	Salve as alterações.
As modificações ficarão disponíveis imediatamente após a confirmação.

4.9 Ativação e Desativação de Contas
Quando necessário, um usuário poderá ser desativado sem que seus dados sejam excluídos.
Conta ativa
•	permite acesso ao sistema.
Conta inativa
•	impede novos logins;
•	preserva o histórico do usuário;
•	mantém registros relacionados para auditoria.
A reativação poderá ser realizada a qualquer momento por um usuário autorizado.

4.10 Redefinição de Senha
Caso seja necessário, um administrador poderá redefinir a senha de um usuário.
Fluxo simplificado:
Selecionar Usuário
        │
        ▼
Redefinir Senha
        │
        ▼
Nova Senha Temporária
        │
        ▼
Primeiro Login
        │
        ▼
Alteração Obrigatória da Senha
Após o primeiro acesso, o usuário deverá criar uma nova senha pessoal.

4.11 Exclusão de Usuários
Por motivos de rastreabilidade, o sistema poderá utilizar desativação lógica em vez de exclusão física.
Isso significa que:
•	o histórico permanece preservado;
•	registros relacionados continuam íntegros;
•	auditorias não são comprometidas.
A exclusão definitiva, quando permitida, deverá obedecer às políticas definidas pela administração do sistema.

4.12 Auditoria

•	Alterações salvas.
•	Senha atualizada.
•	Relatório gerado.
•	Registro excluído com sucesso (quando permitido).
Essas mensagens geralmente desaparecem automaticamente após alguns segundos.

8.5 Mensagens de Aviso
As mensagens de aviso indicam situações que merecem atenção, mas que não impedem necessariamente a continuidade da operação.
Exemplos:
•	Existem campos obrigatórios não preenchidos.
•	O território já está atribuído a outro publicador.
•	O relatório contém grande quantidade de registros.
•	Existem alterações não salvas.
O usuário deve analisar o conteúdo da mensagem antes de prosseguir.
8.6 Mensagens de Erro
As mensagens de erro são apresentadas quando uma operação não pode ser concluída.
Exemplos:
•	Usuário ou senha inválidos.
•	Acesso não autorizado.
•	Não foi possível salvar os dados.
•	Erro de comunicação com o servidor.
•	Registro não encontrado.
Sempre que possível, o sistema apresentará orientações para resolução do problema.

8.7 Notificações de Segurança
O GTP poderá emitir notificações relacionadas à segurança da conta.
Exemplos:
•	alteração de senha realizada;
•	tentativa de acesso inválida;
•	sessão expirada por inatividade;
•	login realizado em novo dispositivo;
•	redefinição de credenciais.
Esses avisos ajudam o usuário a identificar possíveis acessos não autorizados.

8.8 Notificações Operacionais
São mensagens relacionadas às atividades do dia a dia da congregação.
Exemplos:
•	nova designação disponível;
•	território devolvido;
•	território aguardando distribuição;
•	publicação cadastrada;
•	atualização de cadastro;
•	tarefa pendente.
Essas notificações facilitam o acompanhamento das atividades operacionais.

8.9 Notificações Administrativas
Administradores poderão utilizar o sistema para divulgar comunicados importantes.
Exemplos:
•	manutenção programada;
•	atualização do sistema;
•	alteração de procedimentos;
•	avisos gerais;
•	comunicados internos.
Essas mensagens poderão ser direcionadas para todos os usuários ou apenas para determinados perfis.

8.10 Histórico de Notificações
Quando disponível, o sistema manterá um histórico das notificações recebidas.
O usuário poderá:
•	visualizar mensagens anteriores;
•	identificar notificações lidas e não lidas;
•	consultar a data e hora do recebimento;
•	acessar detalhes da mensagem.
Esse recurso facilita o acompanhamento de eventos importantes.

8.11 Indicadores Visuais
As notificações poderão ser identificadas por elementos visuais, como:
•	ícones;
•	cores;
•	indicadores numéricos;
•	marcação de mensagens não lidas.
Esses recursos ajudam o usuário a reconhecer rapidamente a prioridade de cada aviso.

8.12 Confirmação de Operações
Algumas ações críticas exigirão confirmação antes da execução.
Exemplos:

9 – Perguntas Frequentes e Solução de Problemas
9.1 Objetivo
Este capítulo reúne as dúvidas mais frequentes dos usuários do Gestor de Territórios e Publicações (GTP) e apresenta orientações para solucionar os problemas mais comuns encontrados durante a utilização do sistema.
O objetivo é permitir que o usuário resolva situações recorrentes de forma rápida, reduzindo a necessidade de suporte técnico e promovendo uma experiência mais eficiente.

9.2 Antes de Solicitar Suporte
Antes de entrar em contato com o administrador ou com a equipe de suporte, recomenda-se verificar:
•	se a conexão com a internet está funcionando corretamente;
•	se o navegador está atualizado;
•	se a página foi recarregada;
•	se o problema ocorre apenas em uma funcionalidade específica;
•	se há alguma mensagem de erro exibida pelo sistema;
•	se o usuário possui permissão para executar a operação desejada.
Muitas situações podem ser resolvidas com essas verificações iniciais.

9.3 Problemas de Acesso
Não consigo fazer login.
Possíveis causas:
•	usuário ou senha incorretos;
•	conta desativada;
•	senha expirada;
•	erro de digitação;
•	indisponibilidade temporária do sistema.
Solução:
•	verifique os dados informados;
•	utilize a opção Esqueci minha senha;
•	confirme com o administrador se sua conta está ativa;
•	tente acessar novamente após alguns minutos, caso haja manutenção.

Esqueci minha senha.
Utilize a opção Esqueci minha senha na tela de login.
Caso não receba o e-mail de recuperação:
•	confirme se o endereço eletrônico cadastrado está correto;
•	verifique a pasta de spam ou lixo eletrônico;
•	entre em contato com o administrador, se necessário.

Minha conta foi bloqueada.
O bloqueio pode ocorrer por:
•	excesso de tentativas de login;
•	políticas de segurança;
•	desativação da conta.
Nessa situação, solicite a liberação da conta ao administrador responsável.

9.4 Problemas de Navegação
A página não carrega.
Verifique:
•	conexão com a internet;
•	disponibilidade do servidor;
•	funcionamento do navegador;
•	existência de atualizações pendentes.
Se o problema persistir, tente acessar utilizando outro navegador compatível.

Algumas informações não aparecem.
Isso pode ocorrer devido a:
•	permissões insuficientes;
•	filtros aplicados;
•	ausência de registros;
•	atualização pendente da página.
Revise os filtros e atualize a consulta.

O sistema está lento.
Possíveis causas:
•	conexão lenta;
•	grande volume de dados;
•	manutenção do servidor;
•	elevado número de acessos simultâneos.
Aguarde alguns instantes e tente novamente.

9.5 Problemas em Cadastros
Não consigo salvar um registro.
Verifique se:
•	todos os campos obrigatórios foram preenchidos;
•	não existem informações inválidas;
•	não há registros duplicados;
•	sua conta possui permissão para realizar a operação.
Leia atentamente a mensagem apresentada pelo sistema.

O registro desapareceu da lista.
Possíveis causas:
•	aplicação de filtros;
•	alteração do status;
•	exclusão lógica;
•	mudança de página na listagem.
Remova os filtros e realize uma nova pesquisa.

Não consigo editar um registro.
Isso pode ocorrer quando:
•	o usuário não possui permissão;
•	o registro está bloqueado por regras do sistema;
•	a operação é restrita ao administrador.

9.6 Problemas com Relatórios
O relatório não apresenta informações.
Verifique:
•	período selecionado;
•	filtros aplicados;
•	existência de dados cadastrados;
•	permissões de acesso.

Não consigo exportar o relatório.
Possíveis causas:
•	funcionalidade indisponível para seu perfil;
•	bloqueio do navegador;
•	falha temporária na geração do arquivo.
Tente novamente após alguns instantes.

A impressão ficou incorreta.
Antes de imprimir:
•	utilize a visualização de impressão;
•	confirme a orientação da página;
•	ajuste as margens, se necessário;
•	selecione a impressora correta.

9.7 Problemas com Notificações
Não recebo notificações.
Verifique:
•	configurações da conta;
•	e-mail cadastrado;
•	permissões do perfil;
•	conexão com a internet.
